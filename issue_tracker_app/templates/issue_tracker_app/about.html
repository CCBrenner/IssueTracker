About Issue Tracker

Welcome to Issue Tracker, a tool for keeping project to-do list items tracked, organized, and accessible. This issue tracker was designed to be used for software development feature implementation and bug tracking contexts but it can be applied to other contexts too such as any kind of personal project, business project, customer service issue list, as a task manager, and many more. Issue Tracker is free for small projects and demonstration purposes and may be expanded for future purposes (see Production and Purpose below).


How 'Issue Tracker' is Designed

This app was designed to be intuitive to learn through using and navigating between pages. The navigation bar highlights the section of the app that you are currently in and then changes when you move to a new location within the app. It is structured in a three-part hierarchy where the user creates projects followed by creating issues within a specific project. You can always return to the parents and grandparents of the current location you are in by clicking on the navigation item to the left of where your are but to view or work on children and grandchildren of your current location, you must choose which child or grandchild you would like to go into within the current page, taking you further down the hierarchy. You'll see the Logout as left-most (save for the app title) which is then followed by Profile, Project, and Issue as the three levels of the app.

Technologies Used

Issue Tracker utilizes Python, Django, HTML5, CSS3, W3.CSS, vanilla JavaScript, PostgreSQL, and Heroku Linux server (Dyno). It was developed in Django which uses the Python programming language and MVT architectural design pattern is required as a part of the Django framework (similar to the MVC design pattern). MySQL was the development database and PostgreSQL was chosen as the production database. A large majority of styles implemented are made possible by W3.CSS responsive classes which can be described as a simpler version of Twitter Bootstrap that does not use Jquery and which has an overall clean, paper-like appearance. CSS3 is used for footer placement and various fonts within the largely altered W3.CSS responsive base template. JavaScript was used for sorting functions on rendered table data. Heroku's uniquely designed Linux servers called Dynos are being used for hosting.


Production and Purpose

'Issue Tracker' was created during a 6-week class over the span of less than 5 weeks as a capstone project by Collyn Christopher Brenner to fulfill two purposes. The first of these was to submit as a capstone project in finishing up his undergrad as a Computer Information Technology major. The second purpose was to be exactly what the class intended the project to be: A portfolio artifact to not just tell but to show the kind of knowledge and skills Collyn had acquired while in school at Trevecca Nazarene University.
